## Key Design Principles

1. **Separation of Concerns**
   - Each component has a single responsibility
   - Handlers handle source-specific logic
   - Platform handlers handle platform-specific logic

2. **Dependency Inversion**
   - High-level services depend on abstractions
   - Platform handlers implement the interface
   - Easy to swap implementations

3. **Open/Closed Principle**
   - Open for extension (new handlers, platforms)
   - Closed for modification (core services unchanged)

4. **Factory Pattern**
   - PlatformFactory creates platform instances
   - Handler registry maps types to classes
   - Singleton pattern for platform handlers

5. **Configuration-Driven**
   - All behavior controlled via YAML
   - No hardcoded platform dependencies
   - Easy to switch platforms
